# Skills Folder

Store all reusable skill documents in this folder.

## Conventions
- One skill per Markdown file.
- Use clear names, for example: `step_result_unify_skill.md`.
- Keep skills focused and executable (rules, examples, constraints).

## Current Skill
- `step_result_unify_skill.md`: Used to standardize `step` and `expected_result` text from parsed TestRail JSON.
